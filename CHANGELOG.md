# 소리노트(Sorinote) 변경 이력 (CHANGELOG)

본 문서는 소리노트(Sorinote) 프로젝트의 업데이트 및 버전별 세부 변경 사항을 기록하는 문서입니다.

---

## v1.3.0 (2026. 06. 11)

### 성능 최적화 (Performance Optimizations)
1. **오디오 파형 데이터 프리-렌더링 및 캐싱 (Peak Data Caching)**
   * **구현**: 오디오 파일 업로드 시 백엔드 단에서 `ffmpeg` 및 `ffprobe`를 이용해 오디오 파형의 Peak 데이터와 전체 재생 시간(Duration)을 미리 추출하여 SQLite 데이터베이스에 캐싱합니다.
   * **효과**: 클라이언트가 웹페이지에 처음 접속하거나 패드를 로딩할 때, 용량이 큰 오디오 파일을 전부 다운로드하고 디코딩하는 무거운 프로세스를 피하고 몇 KB 크기의 가볍고 요약된 파형 데이터만을 수신하여 즉시 파형을 렌더링함으로써 CPU 디코딩 부하 및 초기 로딩 지연을 완벽히 해결했습니다.
2. **기존 오디오 리소스 백그라운드 마이그레이션 적용**
   * **구현**: 백엔드 서버 기동 시 기존 데이터베이스 내에 저장된 오디오 항목들 중 파형 캐시가 존재하지 않는 파일을 탐색해, 백그라운드 스레드에서 자동으로 파형을 추출하여 DB에 보완해 주는 마이그레이션(`migrateExistingAudioPeaks`) 코드를 구현했습니다.
3. **2D 캔버스 뷰포트 가상화 (Windowing / Virtualization) 도입**
   * **구현**: 메모 카드 및 연결 라인의 개수가 늘어남에 따라 발생하는 브라우저의 DOM 렌더링 병목을 방지하기 위해 캔버스 가상화를 적용했습니다. 현재 사용자의 브라우저 화면(뷰포트) 영역과 상하좌우 600px의 안전 버퍼 범위를 비교하여, 화면 밖에 완전히 위치한 메모 카드와 SVG 라인은 실제 DOM 트리 렌더링에서 제외(`null` 반환)하도록 조치했습니다.
   * **효과**: 화면 밖의 오디오 플레이어, 텍스트 영역, 이미지 뷰어 등의 무거운 리소스를 렌더링하지 않음으로써 브라우저의 페인팅 연산 렉을 원천 제거하고 줌 및 캔버스 스크롤링 시의 프레임 반응성을 대폭 향상했습니다. (미니맵 렌더링은 전체 개요를 표현해야 하므로 가상화에서 제외하여 정합성 유지)

## v1.0.0 (2026. 06. 09)

### 버그 수정 (Bug Fixes)
1. **iOS 사운드 파일 업로드 호환성 해결**
   * **문제**: 아이폰(iOS Safari) 환경에서 파일 앱을 통해 소리(오디오) 파일을 업로드할 때 파일 선택이 불가능(비활성화)하게 나오던 현상.
   * **해결**: 오디오 입력 요소의 허용 설정(`accept` 속성)에 구체적인 확장자(`.mp3`, `.wav`, `.m4a`, `.caf`) 및 MIME 타입을 명시적으로 지정하여 기기 내 오디오 파일 선택이 원활하게 작동되도록 조치.
2. **사운드 재생 시 오작동(자동 게시) 방지**
   * **문제**: 모바일 메모 작성 화면에서 사운드 파일을 업로드한 후 플레이어 재생 버튼을 누르면 작성 폼이 서버로 전송되어 글이 자동 게시되던 문제.
   * **해결**: 재생 버튼 태그에 `type="button"` 속성을 명시적으로 부여하여 브라우저의 기본 폼 제출(submit) 동작이 유발되지 않도록 차단.
3. **모바일 텍스트 입력창 포커스 시 화면 강제 확대(Auto Zoom) 차단**
   * **문제**: 모바일 환경에서 이름 입력창이나 내용 입력창을 터치하여 키보드가 올라올 때 화면이 보기 불편하게 강제 확대되는 Safari 고유의 동작.
   * **해결**: 텍스트 입력창과 텍스트 영역의 글자 크기를 모바일(768px 이하) 환경에서 최소 `16px` 이상으로 고정하여 강제 줌인을 효과적으로 방지.
4. **모바일 피드 최하단 메모 카드 잘림 현상 해결**
   * **문제**: 모바일 화면에서 스크롤을 끝까지 내려도 가장 아래에 있는 메모 카드의 모서리 아랫부분과 그림자가 화면 경계면에 가려져 잘려 보이던 현상.
   * **해결**: 모바일 피드 컨테이너의 하단 여백(`padding-bottom`)을 `80px`로 여유 있게 늘려 충분한 여백 공간 확보.
5. **모바일 업로드 ALAC(.m4a) 및 CAF(.caf) 오디오 데스크톱 재생 오류 수정**
   * **문제**: Apple 모바일 기기에서 녹음 품질 설정이 '무손실'일 때 생성되는 ALAC(Apple Lossless) 코덱의 `.m4a` 파일이나 `.caf` 오디오 파일이 일부 데스크톱 브라우저에서 재생되지 않던 현상.
   * **해결**: 서버 오디오 업로드 시 `ffprobe`로 코덱을 검사하여 ALAC 파일은 표준 AAC 포맷으로, `.caf` 파일은 표준 `.m4a`(AAC) 포맷으로 서버단에서 자동 트랜스코딩(`ffmpeg`)하여 저장되도록 개선.

### 기능 개선 (Improvements)
1. **모바일 글쓰기 완료 후 최상단 자동 스크롤 기능 추가**
   * 메모 등록을 완료했을 때 화면이 즉시 최상단으로 부드럽게 스크롤되어, 사용자가 자신이 방금 작성한 새 글(피드 최상단 배치)을 빠르게 확인할 수 있도록 유도.
2. **배포 가이드 문서 보완**
   * 깃허브 배포 가이드 문서(`INSTALL.md`) 내에 관리자 기능(`admin`) 사용 방법 및 포트 번호 변경 방법을 새로 수록하여 프로젝트 운영자의 편의를 향상.
3. **관리자 페이지 실시간 DB 및 업로드 용량 모니터링 기능 추가**
   * 백엔드에 `/api/admin/storage-size` API를 개설하여 데이터베이스 파일 및 업로드 디렉토리 크기를 실시간 계산하고, 관리자 페이지 상단에 용량을 실시간 노출 및 갱신할 수 있는 UI 컴포넌트 적용.

## v1.1.0 (2026. 06. 10)

### 버그 수정 (Bug Fixes)
1. **모바일 전환 후 데스크톱 복귀 시 캔버스 좌상단 고정 및 미니맵 영역 뷰포트 찌그러짐 현상 해결**
   * **문제**: 모바일 크기로 화면을 축소하여 데스크톱 컨테이너가 언마운트(unmount)될 때 `ResizeObserver`가 `0` 사이즈를 임시 감지하여 뷰포트 정보(`viewportSize`)를 훼손하던 문제.
   * **해결**: `scrollRefCallback` 콜백 Ref 기반으로 리팩토링하여 observer의 생명주기를 유동적으로 관리하고, unmount나 레이아웃 왜곡 시 크기 `0`이 전달되면 처리를 건너뛰도록 보호 필터 적용. `scrollPosRef`를 사용해 기존 좌표를 안전하게 보관했다가 컴포넌트가 활성화될 때 스크롤 위치를 복원시켜, 화면을 늘렸을 때도 캔버스가 중앙 정렬을 유지하고 미니맵의 붉은 뷰포트가 올바른 크기로 표시되도록 복구.
2. **모바일 버전 전환 시 불필요한 가로 스크롤바 노출 및 화면 잘림 현상 해결**
   * **문제**: 브라우저 너비를 줄여 모바일 모드로 전환했을 때, 세로 스크롤바로 인해 컨테이너 너비 계산 시 `100vw` 속성이 스크롤바 폭을 포함하여 미세한 가로 스크롤을 유발하던 현상.
   * **해결**: 모바일 최외곽 컨테이너의 가로 길이를 `width: 100%`로 명시하고, `body` 요소에 `overflow-x: hidden` 속성을 추가하여 가로 스크롤을 원천 차단.
3. **글 작성/수정 중 닉네임 변경 시 발생하던 메모 수정 잠금(Lock) 고착화 오류 해결**
   * **문제**: 사용자가 메모를 편집 중이거나 새 메모를 작성하는 화면을 열어둔 상태로 상단 패널에서 닉네임/색상을 변경하면, 소켓 서버에서 해당 사용자가 소유한 기존 잠금 정보의 닉네임과 불일치가 일어나면서 본인이 작성한 메모가 타인에 의해 잠긴 것처럼 오인되어 수정/삭제가 불가능해지고 `수정 중: name` 라벨이 유지되던 현상.
   * **해결**:
     * **백엔드 (`join-room` 소켓 핸들러)**: 사용자가 닉네임을 변경하여 `join-room` 메시지를 다시 전송할 때, 소켓 서버 내 `activeLocks` 목록에서 해당 소켓이 획득한 모든 편집 잠금의 닉네임 정보를 새로운 닉네임으로 자동 갱신하고 다른 클라이언트들에게 변경 사실을 동기화하도록 조치.
     * **프론트엔드 (`Board.jsx`)**: 사용자가 직접 메모의 작성을 완료(`handlePublish`), 삭제(`handleDeleteMemo`), 혹은 취소(X 버튼)할 때, 자신의 컴포넌트 내에 남아있던 잠금 상태(`lockedMemos`)에서도 해당 메모 ID에 대한 키를 즉시 로컬에서 비워내어 정렬 동기화가 유지되도록 예외 코드를 반영.

### 기능 개선 (Improvements)
1. **관리자 페이지 설정 개별 저장 기능 도입**
   * 관리자 페이지 상단의 어드민 환경 설정(배경색 및 폰트 색상) 부분에 별도의 `save` 버튼을 추가하여, 하단의 홈페이지 설정 저장 버튼과 분리하고 저장 범위에 맞는 안내 메시지 ("어드민 설정이 저장되었습니다." / "홈페이지 설정이 저장되었습니다.")가 출력되도록 개별화.
2. **관리자 페이지 월드 목록 편집 기능 강화**
   * 월드 목록 편집 패널에 월드 이름(title)을 직접 수정할 수 있는 텍스트 입력 칸(`name` input)을 추가하고, 백엔드의 `PUT /api/pads/:padId` API가 이를 연동하여 데이터베이스에 저장하도록 개선. 색상 선택기 레이블의 모호성을 해결하기 위해 기존 `title` 레이블명을 `title_color`로 명확히 표기.
3. **용량 표시 UI 및 위치 최적화**
   * 실시간 디스크 용량 표시 요소의 불필요한 테두리(border), 안쪽 여백(padding), 배경색을 제거하여 텍스트 형태로 간결하게 노출되도록 스타일 개선. 용량 표시 정보가 복잡한 컬러 픽커와 섞이지 않도록 `admin_bg` 설정 영역의 상단(세로 정렬 구조)으로 배치 변경.
4. **리소스 통계 라벨 영문화 및 메인/어드민 월드 목록 반영**
   * 보드 내의 작성 메모 개수 통계 레이블을 기존 한글(소리, 장면)에서 영문(`sound`, `scene`)으로 통일. 메인 화면과 어드민 화면의 월드 목록 뷰 내의 개별 생성된 월드 URL 하단 부분에도 이 통계 정보(`note`, `sound`, `scene`)가 일치하는 크기로 표시되도록 확장.
5. **데스크톱 버전 월드 목록 레이아웃 가로 정렬 보정**
   * 메인 페이지와 관리자 페이지 내의 월드 목록(데스크톱 기준)에서 날짜, 월드 제목, URL/통계 정보, 주소 복사 버튼의 세로 정렬이 미세하게 어긋나던 문제를 요소별 align-items 조정을 통해 바로잡음.
6. **월드 보드 최초 로드 시 최소 줌 상태로 최적화**
   * 보드 화면에 처음 진입하여 렌더링될 때, 사용자의 해상도 환경에 딱 맞춰 캔버스 전체가 보기 좋게 펼쳐지는 최소 줌(`minZoom`) 상태로 초기화되도록 로직을 수정.

## v1.2.0 (2026. 06. 11)

### 버그 수정 (Bug Fixes)
1. **데스크톱 환경 메모 줄바꿈(엔터) 미적용 현상 해결**
   * **문제**: 데스크톱 화면에서 메모 내용에 엔터(줄바꿈)를 입력해 게시 또는 완료하더라도, 줄바꿈이 무시되고 한 줄로 다닥다닥 붙어서 렌더링되던 현상.
   * **해결**: 데스크톱 메모 카드 본문의 텍스트 출력 `div` 스타일 속성에 `whiteSpace: 'pre-wrap'`을 추가하여 작성된 줄바꿈 기호와 공백이 정상적으로 렌더링되도록 수정.
2. **월드 리스트 날짜 컬럼 및 정보 영역 정렬 오류 수정**
   * **문제**: 날짜와 시간이 표시될 때 고정 너비 제한으로 인해 시간이 아랫줄로 개행(줄바꿈)되던 버그 및 월드 제목 버튼에 설정된 기본 패딩(`padding-left: 8px`)으로 인해 하단의 URL 및 리소스 통계 데이터의 시작 위치가 미세하게 어긋나 보이던 현상.
   * **해결**: `.pad-date` 너비를 `220px`로 상향하고 `white-space: nowrap`을 지정하여 날짜와 시간이 언제나 단일 행으로 함께 출력되도록 보장했습니다. 또한 하단 영역 전체에 일치하는 여백(`padding-left: 8px`)을 추가하여 세로 정렬 라인을 수직으로 통일.

### 디자인 개선 (Design/UX)
1. **메모 카드 구성 요소별 폰트 크기 정밀 조정**
